# 기능 요청 백로그

작성: 제품팀
출처: 제품 보드 + 영업 요청 취합

고객과 영업에서 들어온 "이런 기능 있었으면" 요청을 모아 우선순위를 매기는 위시리스트다. 어디까지나 앞으로 만들면 좋겠다는 바람이지 지금 제품에 대한 불만은 아니다. 설문에서 나온 불만 1위가 궁금하면 이 파일 말고 설문 결과를 봐야 한다.

## 가장 많이 요청된 기능

표 수는 제품 보드와 영업 채널에서 모은 요청 건수고, 매월 중복 정리하고 다시 집계한다.

| 요청 | 표 | 메모 |
|------|----|------|
| Slack 알림 | 64 | 1위. 팀이 이미 쓰는 협업 공간으로 알림을 끌어오자는 요청 |
| 더 많은 차트 종류 | 52 | 대시보드 표현력 넓혀달라 |
| 예약 리포트 이메일 | 47 | 정기 리포트 자동 발송 |
| Salesforce 통합 | 39 | 엔터프라이즈 영업에서 주로 나옴 |
| 다크 모드 | 31 | 표는 많은데 임팩트는 작음 |
| 모바일 위젯 | 22 | 이동 중 핵심 지표 확인용 |

제일 많이 나온 건 64표 받은 Slack 알림이다. 사용자들이 우리 제품을 따로 켜기보다 이미 일하는 공간 안에서 신호를 받고 싶어 한다는 뜻이다.

## 묶어서 보면

상단은 통합(Slack, Salesforce)이 차지한다. 단독 도구가 아니라 기존 스택에 끼워 쓰고 싶다는 얘기다. 예약 리포트 이메일처럼 리포팅을 자동화해달라는 요청도 계속 올라온다. 다크 모드 같은 건 표는 잘 받지만 실제 성과로 이어지진 않는다.

## 우선순위

Slack과 Salesforce는 거래에 직접 걸리는 경우가 많아서, 특히 엔터프라이즈 쪽에서 먼저 손대야 한다. 예약 이메일은 만들기 부담이 적고 원하는 사람이 많아 빨리 쳐낼 후보다. 차트 종류는 그다음, 다크 모드랑 모바일 위젯은 뒤로 미룬다.

## 당분간 안 함

네이티브 모바일 앱 재작성이랑 화이트라벨은 올해 빼기로 했다. 수요는 있는데 투입 대비 효과가 아직 애매하다.

## 프로세스

요청은 월 1회 중복 정리하고 표를 다시 계산한다. 영업은 거래를 막는 요청이면 따로 표시해서 우선 검토로 올릴 수 있다.
