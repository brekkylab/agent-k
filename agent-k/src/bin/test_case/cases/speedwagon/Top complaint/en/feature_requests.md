# Feature Requests Backlog

Source: product board + sales requests
Purpose: ranked wishlist

These are requests for new capabilities — what customers WANT, not complaints
about what exists. A request is not a complaint; don't read this as survey
sentiment.

## Top requested features

| request | votes |
|---------|-------|
| Slack notifications | 64 |
| More chart types | 52 |
| Scheduled report emails | 47 |
| Salesforce integration | 39 |
| Dark mode | 31 |
| Mobile widgets | 22 |

The most requested feature is **Slack notifications (64 votes)**.

## Themes

- Integrations dominate the top of the list.
- Reporting automation is a recurring ask.
- Cosmetic requests (dark mode) are popular but low-impact.

## Prioritization notes

- Slack and Salesforce are revenue-linked; prioritize for enterprise.
- Scheduled emails are cheap and broadly wanted.
- Chart types are a fast follow.

## Not in scope soon

- Native mobile app rewrite.
- White-labeling.

## Process

- Requests are deduped and vote-weighted monthly.
- Sales can flag deal-blocking requests for fast-track review.

## Reminder

This backlog captures desires, not grievances. The survey's most common
complaint is a separate question, answered in the survey results file.
